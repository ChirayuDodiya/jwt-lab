Attack Instructions — How to Bypass via jku Header Injection

The vulnerable endpoint reads the jku URL from the JWT header and fetches the public key from it without validating the domain. You can point it to your exploit server and forge an administrator token.

1 Generate a new RSA key pair in Burp Suite
Go to JWT Editor Keys tab → click New RSA Key → click Generate → OK.
2 Upload your public key to the exploit server
Right-click the new key → Copy Public Key as JWK. Paste it into the exploit server box above (inside the keys array) and click Store on Exploit Server. The exploit URL is:
3 Intercept the GET /api/vulnerable-admin request in Burp
Click the button below, intercept via Burp proxy, and switch to the JSON Web Token tab in Repeater.
4 Modify the JWT header
In the JWT header, change: 
{
  "typ": "JWT",
  "alg": "RS256",
  "kid": "YOUR-KEY-ID",       ← set to the kid of your generated key
  "jku": "http://localhost:PORT/exploit/jwks.json"   ← point to exploit server
}
5 Change the sub claim to admin
In the JWT payload, set "sub": "admin".
6 Sign with your RSA private key
Click Sign at the bottom of the JWT tab, select the RSA key you generated, choose Don't modify header, then click OK.
7 Access the vulnerable admin panel
Send the forged request to /api/vulnerable-admin (or click the button below) to solve the lab.
