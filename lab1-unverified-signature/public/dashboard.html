Attack Instructions — How to Bypass JWT Signature Verification

The vulnerable endpoint uses jwt.decode() instead of jwt.verify(). This means the JWT signature is never checked, and you can modify the payload freely.

1 Login with provided credentials
Use chirayu / chirayu123 to login. You'll receive a JWT token.
2 Intercept the request in Burp Suite
Open Burp Suite and configure your browser to use Burp's proxy (127.0.0.1:8080). Navigate to the Vulnerable Admin page and intercept the request.
3 Decode the JWT token
Copy the JWT from the Cookie: token=... header. Go to jwt.io or use Burp's decoder. The payload will look like: 
{
  "sub": "chirayu",
  "role": "user",
  "iat": 1234567890,
  "exp": 1234571490
}
4 Modify the JWT payload
Change "sub": "chirayu" to "sub": "admin" in the payload. Base64url-encode the modified payload and replace it in the token. Keep the same header and signature — the server won't check them! 
// Original payload (base64url decoded):
{"sub":"chirayu","role":"user","iat":...,"exp":...}

// Modified payload:
{"sub":"admin","role":"admin","iat":...,"exp":...}
5 Send the modified request
Replace the cookie value with the modified JWT token and forward the request in Burp Suite. The server will accept it because it uses jwt.decode() which does NOT verify signatures.
