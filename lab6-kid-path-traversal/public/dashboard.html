This HS256-signed token contains a kid header parameter. The server uses kid to locate the signing key on the filesystem.

🔑 How the Server Resolves Keys

The server reads the signing key from the filesystem using the kid parameter: 

// Server code (VULNERABLE):
const keyPath = path.join(KEYS_DIR, kid);
const secret  = fs.readFileSync(keyPath, 'utf8');
jwt.verify(token, secret, { algorithms: ['HS256'] });
⚠ Vulnerability: No sanitization is applied to kid. Path traversal sequences like ../ are accepted as-is, allowing you to read any file on the server — including an empty file whose contents you know.

Attack Instructions — How to Bypass via kid Path Traversal

The server builds a file path from kid without sanitization. Point it to an empty file so the signing secret is an empty string, then sign the JWT with a null byte (AA== in Base64).

1 Login with provided credentials
Use wiener / peter to login. You'll receive a JWT with kid: "secret.key".
2 Generate a symmetric key in Burp Suite
Go to JWT Editor Keys tab → click New Symmetric Key → click Generate. Replace the value of the k property with AA== (Base64-encoded null byte). Click OK.
3 Intercept the request in Burp Suite
Navigate to the Vulnerable Admin page, intercept the GET /api/vulnerable-admin request, and send it to Repeater. Switch to the JSON Web Token tab.
4 Modify the JWT header — path traversal in kid
Change kid to a path traversal pointing to an empty file: 
// On Linux (PortSwigger style):
"kid": "../../../../../../../dev/null"

// In this lab (cross-platform, the lab includes an empty dev/null file):
"kid": "../dev/null"
5 Change the sub claim to admin
In the JWT payload, set "sub": "admin".
6 Sign with the null-byte symmetric key
Click Sign, select the symmetric key you created (with k = AA==), choose Don't modify header, then click OK.
7 Access the vulnerable admin panel
Forward the forged token. The server reads the empty file, gets an empty secret, and your null-byte-signed token matches. Access /api/vulnerable-admin (or click the button below) to solve the lab.
📝 Note: This lab includes an empty file at dev/null (relative to the lab6 directory) for Windows compatibility. Use kid: "../dev/null" to traverse from the keys/ directory to this empty file.
